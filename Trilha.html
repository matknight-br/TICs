Trilha: Do Servidor Tradicional à Orquestração

1. Contextualização

O Ambiente Tradicional: Geralmente, você tem um servidor físico ou uma máquina virtual (ex: um servidor rodando Ubuntu). Você instala todas as dependências do sistema operacional manualmente, como pacotes, bancos de dados e linguagens de programação.

Principais Problemas:

  • Conflito de versões: Uma aplicação exige uma versão específica de uma biblioteca e outra aplicação no mesmo servidor exige uma versão diferente.
  • "Na minha máquina funciona": O ambiente de desenvolvimento é diferente do ambiente de produção.
  • Desperdício de recursos: Criar várias Máquinas Virtuais (VMs) consome muito processamento e memória.

O que é um ambiente orquestrado: Em vez de virtualizar o hardware, empacotamos a aplicação e suas dependências em Containers. O orquestrador (ex: Kubernetes) gerencia esses containers: reinicia falhas, escala recursos e distribui tráfego.

2. Imersão

Situação Problema

Você gerencia um servidor Ubuntu com um sistema legado em Python 2.7. Uma nova API exige Python 3.10. A instalação direta está causando quebras no sistema antigo.

Análise da Infraestrutura

  • Ponto de falha: Acoplamento forte com o SO.
  • Solução: Isolar cada versão em seu próprio container.
